# **베타 테스터 필수 숙지 사항**



### **※베타 테스터는 본 저작물(게임)을 무단 공유하거나 배포하지 말 것※**





**베타 테스터 규칙**



**깃허브 이슈 사용법**(계정 없을 시 배포자 **개인** 디스코드로 보내기)



1. 링크([https://github.com/PKP-Park-Kim-Park/Greed\_Machine\_BugReport/issues](https://github.com/PKP-Park-Kim-Park/Greed_Machine_BugReport/issues))에 접속
2. 오른쪽의 New issue 버튼을 클릭
3. Title에 버그 내용의 키워드 작성(ex. 특정 구간에서의 렉)
4. Description에 아래 \[유저 리포트]를 작성(영상은 10MB 미만 첨부 가능)
5. Create 버튼 클릭하여 작성 완료
6. 깃허브 계정이 없다면 배포자 개인 디스코드로 리포트만 보내기



---

\[유저 리포트]



PC 사양(CPU/GPU/해상도):



발생 시각: (ex. 27일 11시 15분)



버그 내용: (가능한 상세하게) ex. "슬롯머신을 나올 때 차트가 눈 앞에 떠있음."



재현 방법: (어떤 행동을 했을 때 버그가 발생 했는지) ex. "슬롯머신 도중 차트를 여는 동시에 슬롯머신을 나오면 차트가 눈 앞에 떠있음"



스크린샷/영상: (가능한 있으면 첨부)



게임 밸런스: (ex. 아이템 성능, 가격, 적의 난이도, 특정 빌드의 사기성 등)



UX/UI: (인터페이스의 가독성 및 편의성, 튜토리얼 때 설명하면 좋은 것들)



퍼포먼스: (렉 걸리는 구간 또는 시점, 최적화 정도)



게임 플레이 경험: (루즈한 구간 또는 시점)
